# WERT-AX AX엔지니어링팀 조직 GitHub

## Repository 네이밍 규칙

- Repository 이름은 소문자와 하이픈(-)으로 구성되어야 합니다.
- 프로젝트 명에는 프로젝트 성격을 나타내는 접두사와 접미사 태그를 포함해야 합니다.

접두사 접미사 태그는 다음과 같습니다:

> 접두사: 프로젝트 주체와 출처를 나타내기 위한 태그이며 팀 자체 개발 코드와, 컨소시엄 및 협력사 등을 구분하기 위 사용합니다

| 접두사 | 설명 |
|--------|------|
| AX | AX 엔지니어링 팀 자체 개발 코드 |
| CO | 컨소시엄 및 협력사 코드 |

> 접미사: 프로젝트 성격을 나타내기 위한 태그이며 프로젝트의 유형과 목적을 구분하기 위해 사용합니다
> 접미사 태그는 프로젝트 성격이 바뀜에 따라 수정 할 수 있습니다, RND 에서 APP 또는 LIB로 변경하는 경우가 대표적입니다

| 접미사 | 설명 |
|--------|------|
| APP | 애플리케이션 개발 프로젝트 (예: 웹 애플리케이션 Backend, Frontend, 모바일 앱 등, 실제 사용자에게 제공되거나 제공을 목표로 하는 소프트웨어 개발 프로젝트) |
| LIB | 라이브러리 개발 프로젝트 |
| DOC | 문서화 프로젝트 |
| RND | 연구 및 개발 프로젝트 (예: 새로운 기술 연구, 프로토타입 개발, 실험적 프로젝트 등 학문적 성격이 강한 프로젝트) |
